# 🛡️ Antigravity AI 자율 코딩 헌장 (Autonomous Coding Rules)

본 헌장은 Antigravity AI 코딩 어시스턴트가 사용자님과 페어 프로그래밍을 진행할 때, 작성하는 모든 소스코드의 품질을 극대화하고 시스템의 무결성을 수호하기 위해 스스로 정의하고 영구히 준수할 **5대 자율 코딩 수칙**입니다. 향후 작성되는 모든 코드와 리팩토링은 본 헌장의 규칙에 엄격히 입각하여 수행됩니다.

---

## ⚖️ 제 1조: [무결성과 예외 방어선] Defensive & Graceful Degradation
1. **무중단 예외 차단**: 단 하나의 외부 API 통신 실패(네트워크 지연, 세션 만료, 일시 차단 등)가 전체 기산 루프 및 대시보드 마비로 도미노 붕괴되는 것을 원천 차단하기 위해, 모든 API 호출 블록은 독립적인 `try-catch` 안전망으로 엄격히 감싼다.
2. **우아한 성능 강하 (Graceful Degradation)**: 필수 자격증명(API Key 등)이 누락되었거나 통신 지연이 발생할 경우, Null Pointer 에러를 뱉으며 죽는 대신 사전에 조율된 **Mocking 데이터 제공** 또는 **네이버 금융 크롤러 등 Fallback 백업 메커니즘**을 즉시 가동하여 시스템을 살려낸다.
3. **자격증명 보호벽**: `settings` 시트 등에서 설정을 서버로 벌크 동기화할 때, B열이 비어있거나 마스킹된 칸(`******`)은 기존에 서버에 기입되어 있던 소중한 Properties 기밀값을 덮어써서 파괴하지 않도록 동기화 대상에서 생략(건너뛰기) 처리한다.

## 🚀 제 2조: [검증 우선 원칙] Static Parsing & VM Sandbox Testing
1. **선제적 정적 구문 검사**: 모든 변경 코드는 배포(Deployment) 혹은 커밋 전에, 정적 파싱 무결성 검증기(Static Syntax Parser)를 로컬에서 선제 기동하여 단 하나의 문법적 치명 오류(Syntax Error)도 프로덕션 서버에 업로드되지 않도록 완전 차단한다.
2. **샌드박스 동적 유닛 테스트**: 주요 주가 정규화, 가중평균 자가 치유 병합 엔진 등 핵심 비즈니스 로직은 배포 전에 반드시 로컬 VM 샌드박스로 모킹 DB를 돌려 유닛 테스트(Dynamic Unit Test) 통과 여부를 100% 검증한다.

## 🎨 제 3조: [프리미엄 비주얼 디자인] Premium UI/UX & HSL Tailored Hues
1. **기본 브라우저 스타일 배격**: Plain red, plain blue 등 browser default 및 투박한 원색 계열을 전면 배제하고, Harmony HSL 기반의 세련된 다크 글래스모피즘(Glassmorphism) 테마를 채택한다.
2. **글래스모피즘 & 네온 아우라**: 반투명 글래스 패널, Subtle Glowing(네온 아우라) 테두리 칩 효과, smooth gradients, 그리고 Inter/Outfit 구글 폰트를 일관되게 조합하여 Wowing 비주얼을 제공한다.
3. **마이크로 애니메이션 피드백**: 모바일 터치 및 버튼 마우스 호버 시 스케일 축소/확장 트랜지션을 부드럽게 이식하여 인터페이스가 Responsive하고 Alive하게 살아 움직이도록 설계한다.

## 📝 제 4조: [품격 있는 한글 소통 및 문서 보존] Respectful Korean & Doc Integrity
1. **정중한 품격 한글**: 모든 사용자 대화, 리포트 발행, 그리고 소스코드 내부 주석은 신뢰성과 전문성이 느껴지는 단정하고 격조 높은 한국어로만 친절하게 기술한다.
2. **한국어 공식 답변 절대 수호 (100% Korean responses)**: 어떠한 상황에서도 영어로 답변이 새어나가는 현상을 원천 차단한다. 사용자에게 출력되는 모든 메시지, 보고서, 안내글은 100% 깔끔하고 자연스러운 한글만을 사용하여 최종 소통한다.
3. **사고 과정(Thought Process) 한글화 절대 수호**: AI 모델의 내부적인 독백 및 추론 영역(사고 과정 태그 본문) 또한 오직 한글만을 사용하여 생각하고 기획한다. 생각하는 모든 단계에서 영어 사용을 전면 배제하고 철저하게 한국어만을 사용하여 언어 일관성을 100% 준수한다.
4. **레거시 문서 및 주석 존중**: 코드 리팩토링이나 기능 추가 시, 기존에 작성되어 있는 레거시 비즈니스 주석, 작성 기록, 저작권을 함부로 소거하지 않고 온전히 보존하며 그 정합성을 확장 수호한다.
5. **자율적 주간 갱신**: 개발 전에는 `implementation_plan.md`를 통해 기획을 공유받고, 개발 도중에는 `task.md`를 갱신하며, 개발 완료 후에는 `walkthrough.md`로 보고하는 3대 아티팩트 라이프사이클을 영구히 준수한다. (각 경로는 현재 대화 세션의 아티팩트 보관용 고유 디렉토리 절대 경로를 동적으로 링크 처리하여 활용한다.)

## ⚙️ 제 5조: [멱등성 및 환경 대응] Idempotency & Clean Workspace
1. **멱등성(Idempotency) 준수**: 동일한 기산 함수나 초기화 트리거를 10번 실행하더라도 데이터가 중복 적재되거나 꼬이지 않고, 항상 동일하게 정화(Clean up & Weighted Average)된 단일 무결 상태를 보장하도록 데이터 멱등성을 실현한다.
2. **초경량 워크스페이스 보존**: 불필요한 레거시 임시 백업 시트, 임시 파일들은 정기 대청소 메뉴(`menuCleanupLegacySheets`) 등을 활용하여 수시로 제거함으로써, 클라우드 리소스를 100% 쾌적하고 슬림하게 관리한다.

## ⚡ 제 6조: [자율 즉시 실행 및 무중단 배포] Full Autonomy & Non-Stop Pipeline
1. **승인 대기 생략 및 자율 실행**: 유저님의 전폭적이고 강력한 자율 코딩 권한 위임에 입각하여, 개선 요구에 대해 스스로 완벽히 판단하여 기획 및 설계를 수행한다.
2. **정밀 자율 준비 및 협력적 계획 승인 (High Autonomy & Collaborative Planning)**: 기획안의 자율 설계와 코딩 구현 및 로컬 테스트 검증은 고도의 전문성으로 스스로 주도하여 완료하되, 구현 착수 전에 **구현 계획서(implementation_plan.md)를 통해 사용자에게 계획을 안내하고 조속히 계획 승인(Approve)을 득한 후 실행 단계로 진입**하여 개발 안전성을 영구 수호한다.
3. **무중단 CI/CD 배포 파이프라인**: 설계된 내용은 자율적으로 `task.md` 에 기록하고, 단위 테스트 검증을 선제 수행한 뒤, clasp 배포 및 실시간 모니터링 검증까지 무중단 원스톱(Non-Stop)으로 완벽하게 마무리하여 결과 보고서(`walkthrough.md`)와 함께 최상의 상태를 즉각 대령한다.

## 🛡️ 제 7조: [오류 예방 및 자가 정화 수칙] Bug Prevention & Self-Healing
1. **정규화 데이터 유실 방지 (No Variable Leakage)**:
   - 데이터 정규화 변수(예: `sym = normalizeStockSymbol_(row.symbol)`)를 선언하고 검증했다면, 이후의 API 호출, Fallback 검색, 로그 기록, 그리고 최종 데이터베이스(시트) 적재 객체 생성 시 절대 원래의 원시 오염 데이터(`row.symbol` 등)를 참조하거나 유출하지 않고 **오직 정규화된 변수만을 일관되게 끝까지 참조**하도록 이중 확인한다.
2. **백엔드 물리적 자가 치유 (Self-Healing Overload)**:
   - 데이터베이스나 스프레드시트의 원본 데이터가 앞자리 `0` 증발, 숫자로 오염된 종목명(`660` 등)과 같이 꼬인 상태로 적재된 것을 감지할 경우, 단순히 뷰 레이어에서 일회성 보정으로 때우지 않는다.
   - 백엔드가 구동되는 즉시 실시간 KIS/Naver 조회를 통해 올바른 한글명("SK하이닉스")을 스스로 획득한 뒤, **스프레드시트 원본 셀의 데이터를 올바른 한글 및 정격 코드로 즉시 물리적 덮어쓰기 정화(Self-Correction)**를 단행하여 유저가 데이터를 수동으로 삭제하고 재등록하는 불편함을 원천 박멸한다.
3. **캐시 고착화(Cache Poisoning)의 즉각적 파괴**:
   - 자산의 수정, 추가, 삭제 등 데이터의 구조적 변경 이벤트가 발생하는 API 블록에서는 갱신 속도 튜닝용 캐시(예: `collectHoldingsCurrent(false)`)를 절대 맹신하지 않는다.
   - 변경 즉시 메모리 상의 포트폴리오 잔고 캐시(`KIS_REAL_PORTFOLIO_CACHE` 등)와 한글명(`AM_NAME_`), 가격 캐시(`AM_PRICE_`)를 남김없이 폭파(`remove`) 시키고, `collectHoldingsCurrent(true)` 로 강제 동기화하여 캐시 고착화가 일어나지 않도록 철저히 영구 수호한다.
4. **반응형 찌그러짐 차단 (Robust Flexible Layout)**:
   - 모바일 하이브리드 미니앱 환경에서 소수점 수량, 긴 텍스트 등이 유동적으로 인입되는 영역의 인접 레이아웃 요소(예: 은행/증권사 라벨 배지 등)는 너비 압착 방지(`flex-shrink: 0`) 및 글자 줄바꿈 방지(`white-space: nowrap`) 속성을 필수 장착하여, 좁은 화면에서도 텍스트가 잘려 개행되거나 배지가 찌그러지는 레이아웃 붕괴를 사전에 완전 차단한다.
5. **로컬 실행 정책 및 쉘 환경 제한 우회 규정 (CI/CD Bypass)**:
   - 로컬 개발 환경에서 clasp CLI 푸시/배포 명령 수행 시 Windows PowerShell 등 운영체제의 실행 정책 제한(`PSSecurityException`)으로 원격 동기화가 차단되는 대참사가 없도록, 배포 명령어는 항상 권한 우회 옵션(`powershell -ExecutionPolicy Bypass`)을 표준화하여 CI/CD 파이프라인의 안전 전송률을 100% 사수한다.

## 💎 제 8조: [이중 Fallback과 프론트 안전 정화막] Double-Fallback & Double-Shielding
1. **0원/0% 방치 오류 차단 및 이중 Fallback 시세망**:
   - 외부 가격 조회 API가 한국 주간 점검, 장외 휴장일, 세션 만료로 일시 실패할 경우, 단순히 평단가(`avg`)를 현재가로 덮어쓰며 무마하지 않는다. 이 경우 손익금액이 `0원`이 되고 수익률/등락률이 모두 `0`에 얼어붙어 유저를 극도로 불안하게 만드는 비주얼 참사가 발생하기 때문이다.
   - 따라서 API 실패 시 차선책으로 돌며 실시간 진짜 가격과 전일 대비율을 100% 무조건 스캔 및 파싱해오는 **'네이버 금융 Fallback 등 이중 가격 스캔망'**을 백엔드에 연쇄 장착하여, 어떠한 상황에서도 신선한 실시간 시세와 등락 피드를 안전하게 적재한다.
2. **프론트엔드 종목명 이중 치환벽 (Double-Shielding)**:
   - 스프레드시트 적재 꼬임이나 백엔드 서버 캐시 고착화 지연으로 임시 티커명이나 오염된 문자가 날아오는 현상을 화면 단에서 완벽하게 차단하기 위해, 프론트엔드가 데이터를 수령하자마자 메모리 선상에서 교정된 한글 이름으로 치환하는 **'프론트엔드 0초 전처리 치환 파이프라인'**을 상시 기동한다.
   - 이를 통해 F5 새로고침 지연이나 서버 DB 상태와 아예 물리적으로 무관하게, 보유 주식 카드, 상세 손익 팝업, 비중 차트 등 화면의 전 구역에서 영구적으로 티커가 깨끗한 한글명("알파벳 C (구글 C)" 등)으로 100% 보정되어 출력되도록 철벽 방어한다.
3. **리포트 렌더링 내 마크다운 코드블록 완전 정제 및 HTML 이중 정화**:
   - AI가 생성하는 리포트나 퀀트 자문서를 화면 내 네온 대시보드 패널에 렌더링할 때, 유저에게 불필요한 마크다운 백틱(`` ` ``)이나 ` ```html `, ` ``` ` 등 코드 블럭 찌꺼기 선언문들이 지저분하게 날것으로 노출되어 가독성을 방치하지 않는다.
   - 렌더링 직전에 정규식(Regex)을 통해 모든 코드 껍데기 기호들을 완전히 청소 및 스크래핑하고, 오직 가독성 높은 우아한 줄바꿈과 볼드(Bold) 폰트 강조만을 뿜어내도록 리포트 정제 필터를 필수로 적용한다.
   - 동시에 JSON 통신 중 이스케이프되어 날아오는 HTML 엔티티 문자(`&lt;`, `&gt;`, `&amp;` 등)를 진짜 HTML 태그 괄호로 완벽하게 복원하는 **'이중 엔티티 정화 필터(Entity Unescape)'**를 필수 구현하여, 화면상에 태그 문자 코드가 날것으로 오염 노출되는 현상을 원천 차단한다.
4. **그리드 대칭 기반 탭 오터치 방지**:
   - 미니앱 탭 개수가 늘어나 1줄 레이아웃에 찌그러지고 빽빽하게 뭉개질 때는, 억지로 우겨넣지 않고 **'그리드 대칭 기반 2줄 레이아웃'**으로 전격 개편한다.
   - 윗줄과 아랫줄이 대각선 대칭을 이루며 빈틈없이 정렬되는 그리드 비율(예: 6열 중 span 2씩 3개, span 3씩 2개)을 설계하여, 여유롭고 수려한 시각적 HUD 그립감과 오터치 방지 인터랙션을 보증한다.

## 🔁 제 9조: [순환 재귀 방지 및 시세/룩업 레이어 엄격 분리] Circular Recursion Prevention
1. **상호 참조(Circular Reference) 금지**:
   - 시세 조회 코어 함수(`fetchKisOverseasCurrentPrice_`, `fetchYahooOverseasCurrentPrice_`, `fetchKisCurrentPrice_` 등) 내부에서는 절대 한글명 룩업 헬퍼 함수(`getStockKoreanName_`)를 직접 또는 간접적으로 호출하여 상호 참조가 발생하는 일을 차단한다.
   - 룩업 헬퍼 함수(`getStockKoreanName_`) 내부에서는 미등록/오염 이름의 자동 복원을 위해 최후의 보루로 시세 조회를 찔러보도록 설계되어 있기 때문에, 시세 조회 내부에서 다시 이 룩업 헬퍼를 불러버리면 완벽한 무한 루프(재귀 스택 오버플로우)가 형성되기 때문이다.
2. **시세 조회 코어의 순수성 수호**:
   - 시세 조회 API 및 Fallback 스크래퍼 계열은 오직 **현재가, 등락률, 거래량 등 API 원시 응답을 정밀하게 구조화하여 반환**하는 독립적인 코어 역할에만 한정한다.
   - 한글명 룩업 및 정화 가공은 시세 조회가 완전히 종료된 이후의 상위 기산 레이어(`collectHoldingsCurrent` 의 매핑, API 웹 반환부 등)에서만 결합하여 순차적으로 수행한다.
   - 이를 통해 원천적인 네트워크 교착 상태 및 무한 재귀 호출로 인해 모바일 앱이 **'무한 로딩(Infinite Loading)'** 에 빠지는 장애를 영구적으로 예방한다.

## 🛠️ 제 10조: [구글 샌드박스 호환성 및 실무 튜닝 수칙] Google Sandbox & Performance Optimization
1. **중첩 템플릿 리터럴(Nested Backticks) 전면 배제**:
   - 구글 Apps Script의 HTML Service는 서버사이드 전처리기(Caja Parser)를 거쳐 렌더링되므로, 스크립트 내부에서 백틱 안에 백틱이 중첩되는 문법(예: `` `... ${`...`} ...` ``)을 해석하지 못하고 컴파일 및 JS 기동을 마비시킬 수 있다.
   - 이에 따라 동적 HTML 블록을 생성할 때는 절대로 중첩 백틱 구조를 작성하지 않으며, 안쪽 템플릿은 반드시 상위 스코프에서 개별 변수로 쪼개어 가공(예: `srimRowHtml` 단일 가공)한 뒤 최종 결합한다.
2. **비공개 헬퍼 함수 접미사(`_`) 필수 적용**:
   - Google Apps Script 환경은 별도의 파일 스코프 격리가 존재하지 않고 전역 영역을 공유한다.
   - 외부 라이브러리 참조나 자동 실행 트리거 대상에서 제외되어야 하는 프로젝트 내부의 독립 유틸리티 및 헬퍼 함수는 이름 뒤에 반드시 언더바(`_`)를 붙여(예: `helperFunction_`) 외부 노출을 원천 캡슐화 차단한다.
3. **파일명 2자리 넘버링 구조화**:
   - 대규모 스크립트 파일이 얽힌 환경에서 로딩 및 상호 참조 의존성을 직관적으로 구조화하기 위해, 프로젝트 루트의 모든 파일 이름에는 반드시 두 자리 숫자 접두사(예: `00_Config.js`, `16_ReportBuilder.js`)를 명명하여 관리한다.
4. **비동기 RPC 패킹 통신 및 2중 캐싱 레이어**:
   - 클라이언트에서 `google.script.run`을 호출할 때 발생하는 오버헤드를 막기 위해, 대용량 계산 및 데이터 조회 시 단일 호출을 지양하고 데이터를 배열/객체로 묶어서(Packing) 단 1회의 비동기 RPC로 수령한다.
   - 또한, 서버단(CacheService 등)과 클라이언트 메모리에 2중 캐싱 레이어를 상시 도입하여 로딩 속도를 최적화한다.
5. **배포 전 HTML 스크립트 정적 파싱 검증 의무화**:
   - 배포(`clasp push`) 전에 반드시 `.js` 파일 정적 분석뿐 아니라, `index.html` 내부에 위치한 `<script>` 소스코드들만 별도 추출하여 Javascript 문법적 오류가 없는지 검증하는 문법 스캐너(`check_html_js.js`)를 로컬에서 필수적으로 가동한다.

## 🌐 제 11조: [이원화 호스팅 및 클라우드 캐시 우회 수칙] Dual-Hosting Sync & Cache-Busting
1. **GitHub Pages & Apps Script 이원화 동기화 의무**:
   - 대시보드 화면(HTML/CSS/JS)은 GitHub Pages에서 정적으로 서빙되고 백엔드는 Apps Script Web App API로 굴러가는 이원화 구조를 완벽하게 인지한다.
   - 화면 레이아웃, CSS 스타일, 혹은 프론트엔드 JS 연동 로직을 수정한 경우, `clasp deploy`를 통한 Apps Script 배포뿐만 아니라 **반드시 GitHub 원격 저장소(`main` 브랜치)로 즉각 `git push`를 실행**하여 GitHub Pages 웹사이트가 0초 만에 실시간 빌드되어 동시 갱신되도록 배포 생태계를 엄격히 일치시킨다.
2. **구글 CDN 캐시 고착화 파괴 (Cache-Busting Deployment)**:
   - clasp deploy로 기존 배포 ID에 버전을 덮어쓸 때, 구글의 프록시 CDN 캐싱 및 익명 권한 지연으로 인해 실서버가 낡은 HTML/JS를 계속 서빙하는 지연이 관측될 경우, 이를 억지로 기다리지 않는다.
   - 지체 없이 **신규 배포 ID(New Deployment ID)를 생성하여 구글 캐시를 완전히 깨뜨리고**, 새 배포 ID를 `run_test_and_deploy.js`, `index.html` 의 `API_URL`, 그리고 `04_TelegramBot.js` 의 `WEB_APP_URL` 및 `verifiedUrl` 등 3대 연동 경로에 일제히 갱신 적용하여 배포 즉시 100% 최신 패치 코드가 실시간 동작하도록 강제 조치한다.

## ⚡ 제 12조: [초고속 모드 전환 및 대시보드 링크 영구 고정] Instant Swapping & Default Fallback
1. **초고속 모드 스위칭 (Instant Mode Swapping UX)**:
   - 모의투자 <-> 실전투자 모드 토글 전환 시, 무거운 KIS API 실시간 동기화(`collectHoldingsCurrent`)를 동기식으로 무한 대기하게 만들면 5~8초 렉 및 타임아웃이 발생한다.
   - 백엔드는 모드 변경 시 오늘 조회했던 해당 모드의 데이터 스냅샷이 시트에 이미 존재한다면, KIS API 스캔 대기 시간을 전면 생략하고 **0.2초 만에 캐시 데이터로 즉시 응답**한다.
   - 프론트엔드도 모드 변경 성공 시 1.2초 딜레이 대신 **300ms의 최소 대기 후 캐시 기반 고속 갱신(`fetchPortfolioData(false)`)을 즉각 수행**하여 스위치를 클릭하자마자 0.5초 이내에 모드 전환 렌더링이 완료되도록 튜닝한다. KIS API 강제 실시간 조율은 사용자가 수동 새로고침 버튼(`fetchPortfolioData(true)`)을 직접 터치할 때만 한정 가동한다.
2. **대시보드 깃허브 링크 영구 고정 (TMA GitHub Fallback)**:
   - 텔레그램 봇방에서 대시보드 바로가기 링크를 발송할 때, 구글 웹앱 주소로 폴백이 일어나면 권한 오버헤드와 브라우저 경고 배너가 발생하므로 대시보드의 실질적 진입 장벽이 높아진다.
   - 대시보드 URL 반환 함수(`getWebAppUrl_`)는 설정에 별도 `CUSTOM_DASHBOARD_URL`이 기입되어 있지 않거나 동기화가 풀려 있더라도, 기본 구글 웹앱 주소로 폴백하는 대신 **무조건 사용자의 GitHub Pages URL (`https://taewoonghwang.github.io/jusik/`)로 안전 강제 폴백**되도록 설계하여 챗봇을 통한 대시보드 기동 시 항상 쾌적한 깃허브 환경을 보장한다.
